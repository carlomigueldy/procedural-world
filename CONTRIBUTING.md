# Contributing

## Commit Convention

All commits **must** follow [Conventional Commits](https://www.conventionalcommits.org/):

```
<type>(<scope>): <description>
```

Types: `feat`, `fix`, `docs`, `style`, `refactor`, `perf`, `test`, `build`, `ci`, `chore`.

Descriptions must be lowercase, imperative mood, no period.

## Attribution Rules

- **No LLM model attribution** in code, comments, or commit messages.
- **No AI co-author or co-committer** tags. Commits must list only human contributors.
- Do not include "Generated by ...", "Claude", "Copilot", or similar in code or commit history.

## Branching

- Never commit directly to `main`.
- All changes must go through a pull request.
- Branch from `main`, open a PR, and merge after review.
